Namespace changes

We have changed the PHP namespaces of our plugins to match the (actually not documented anywhere) intention of Joomla's core maintainers about third party extensions. For example, Admin Tools' system plugin has changed its namespace from Joomla\Plugin\System\AdminTools to Akeeba\Plugin\System\AdminTools.

This has two implications:

  1. Joomla! caches the namespaces of core and third party extensions in the autoload_psr4.php file in its cache directory. By default, this file is located in administrator/cache/autoload_psr4.php. Joomla! is supposed to delete and create afresh this file whenever it is updating extensions or itself. However, under some circumstances (e.g. the file or its containing directory cannot be written to by PHP), this may not be possible. You may have to delete that file yourself.
  2. Servers with PHP OPcache —or a similar code cache— enabled may have settings which prevent PHP from “seeing” the changes in the updated files of our extension, or Joomla!'s autoload_psr4.php file. Joomla! tries to reset code caches on extension and core updates. However, the server configuration may prevent this from taking effect. If this happens, you will need to ask your host to reset the OPcache.

Despite us and Joomla! itself taking all reasonable precautions to prevent such a change from causing a problem, some server configurations are outside ours and Joomla!'s control, making it impossible to successfully mitigate problems. If your site stops responding with an error indicating that a class cannot be found follow the advice above, namely:

  • Delete the administrator/cache/autoload_psr4.php file.
  • Ask your host to reset the OPcache, or similar code cache, on your site's server.

Kindly note that this is a one-time change. The namespace of our extensions will not change in the future, unless of course Joomla! itself necessitates a change (nothing like that is, however, planned for any Joomla version released until 2027 at the earliest).
